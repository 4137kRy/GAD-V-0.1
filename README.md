## ✨ Возможности

| Категория | Команды | Примеры |
|-----------|---------|---------|
| **Система** | Выключение, сон, выход | «Джарвис, выключи ПК», «переведи в сон» |
| **Окна** | Развёртывание, переключение | «разверни окно», «альт таб» |
| **Приложения** | Браузер, калькулятор, блокнот | «открой браузер», «закрой калькулятор» |
| **Медиа** | Музыка, громкость, пауза | «включи музыку», «громче», «пауза» |
| **Интернет** | YouTube, почта, GitHub | «перейди на ютуб», «открой гитхаб» |
| **Управление** | Деактивация, остановка | «хватит» (ожидание), «стоп» (выход) |

### 🔹 Ключевые преимущества

| Особенность | Описание |
|-------------|----------|
| 📶 **Офлайн-работа** | Распознавание через Vosk локально, без отправки данных в облако |
| 🔤 **Исправление опечаток** | RapidFuzz корректирует ошибки распознавания (расстояние Левенштейна) |
| 🎯 **Умный парсинг** | Двухэтапный поиск: сначала многословные фразы, потом токены |
| 🔊 **Звуковая обратная связь** | WAV-файлы для подтверждения каждой команды (настраивается в JSON) |
| ⚡ **Два режима работы** | Ожидание (активация по фразе) → Активный (приём команд 15 сек) |
| 🧩 **Модульная архитектура** | Каждый компонент независим, легко расширять без изменения ядра |
| 🚀 **Автономная установка** | Скрипты `firstStart.bat` и `start.bat` для быстрого развёртывания |
| 🎤 **VAD (Voice Activity Detection)** | Энергетический детектор речи отфильтровывает тишину, снижая нагрузку на CPU до 70-90% |

## ⚙️ Быстрый старт

### 1. Требования

| Компонент | Версия | Примечание |
|-----------|--------|------------|
| ОС | Windows 10/11 (64-bit) | Linux/macOS требуют доработки |
| Python | 3.8+ | Проверено на 3.10, 3.11 |
| Микрофон | Любой | Доступ к устройству записи |
| Интернет | Только для установки | Работа полностью офлайн |

### 2. Установка (Автоматическая)

**Шаг 1: Запустите первичную настройку**

Дважды кликните на **`firstStart.bat`** или выполните в терминале:

```batch
firstStart.bat
```

**Скрипт автоматически:**
- ✅ Проверит версию Python (требуется 3.8+)
- ✅ Создаст виртуальное окружение в `.venv/`
- ✅ Установит все зависимости из `requirements.txt`
- ✅ Создаст папку `models/` для модели распознавания
- ✅ Покажет инструкцию по скачиванию модели Vosk

**Шаг 2: Скачайте модель распознавания**

1. Скачайте: [vosk-model-small-ru-0.22.zip](https://alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip)
2. Распакуйте **содержимое архива** (не сам архив!) в `models/`:

```
models/
└── vosk-model-small-ru-0.22/
    ├── am/
    ├── conf/
    ├── graph/
    └── ...
```

### 3. Запуск помощника

После настройки используйте **`start.bat`**:

```batch
start.bat
```

**Что делает скрипт:**
- ✅ Проверяет наличие виртуального окружения `.venv/`
- ✅ Активирует окружение автоматически
- ✅ Запускает `main.py` через `.venv\Scripts\python.exe`
- ✅ Показывает ошибку если помощник упал

### 4. Первая команда

1. Скажите активационную фразу: **«Джарвис»**, **«эй помощник»**, **«слушай»**
2. После активации (15 секунд) произнесите команду: **«открой браузер»**
3. Помощник выполнит действие и воспроизведёт звук (если настроен)

## ⚙️ Настройка команд

Все команды управляются через **единый файл** `config/commands.json`.

### Структура конфига

```json
{
  "activation_phrases": ["джарвис", "эй помощник", "слушай"],
  "identifiers": {
    "browser": ["браузер", "интернет"],
    "calculator": ["калькулятор", "счёт"]
  },
  "verbs": {
    "open": ["открой", "запусти", "включи"],
    "close": ["закрой", "выключи"]
  },
  "actions": {
    "browser": {
      "open": { "type": "exeStart", "path": "C:\\Program Files\\Firefox\\firefox.exe", "sound_out": "SWO.wav" }
    },
    "calculator": {
      "open": { "type": "exeStart", "path": "calc.exe", "sound_out": "SWO.wav" }
    }
  }
}
```

### 🔹 Типы действий

| Type | Описание | Параметры |
|------|----------|-----------|
| `exeStart` | Запуск приложения | `path` |
| `killProcess` | Завершение процесса | `process` |
| `urlOpen` | Открытие сайта | `url` |
| `fileOpen` | Открытие файла | `file` |
| `mediaPlay` | Воспроизведение медиа | `file` |
| `mediaPause/Next/Prev` | Управление медиа | — |
| `volumeUp/Down/MuteToggle` | Громкость | — |
| `windowMaximize/Minimize/Close` | Управление окнами | — |
| `windowSwitchNext/Prev` | Переключение окон (Alt+Tab) | — |
| `focusWindow` | Активация окна по заголовку | `title_contains` |
| `systemShutdown/Sleep/Logout` | Системные команды | — |
| `assistantDeactivate/Stop` | Управление помощником | — |
| `webSearch` | Поиск в Google | `query` |

### ➕ Как добавить свою команду

**Кратко:**

1. Добавьте идентификатор в `"identifiers"`:
   ```json
   "discord": ["дискорд", "дс"]
   ```

2. Добавьте действие в `"actions"`:
   ```json
   "discord": {
     "open": { "type": "exeStart", "path": "C:\\Discord\\Discord.exe", "sound_out": "SWO.wav" }
   }
   ```

3. Перезапустите помощника (`start.bat`).

**📖 Подробная инструкция:** см. [`docs/ADDING_COMMANDS.md`](docs/ADDING_COMMANDS.md)

В руководстве описано:
- Как работает система команд (архитектура)
- Добавление команд с существующими типами действий
- Создание своего типа действия (программирование)
- Примеры для копирования (email, логирование, запуск скриптов)
- Отладка и валидация JSON

## 🔊 Звуковая обратная связь

### Настройка звуков

Каждое действие может иметь свой звуковой отклик через поле `sound_out`:

```json
"browser": {
  "open": {
    "type": "exeStart",
    "path": "...",
    "sound_out": "browser_open.wav"  // 🔹 Звук воспроизводится после выполнения
  }
}
```

### Требования к файлам

| Параметр | Значение |
|----------|----------|
| Формат | WAV (PCM, 16-bit, 44.1kHz рекомендовано) |
| Расположение | Папка `sounds/` в корне проекта |
| Путь в конфиге | Относительный (`browser_open.wav`) или абсолютный |
| Размер | Рекомендуется до 500 КБ для быстрого воспроизведения |

### Если звук не воспроизводится

1. Проверьте наличие файла в `sounds/`
2. Убедитесь, что имя в `sound_out` совпадает с именем файла
3. Проверьте логи (`logs/assistant.log`) на предупреждения
4. На Windows: убедитесь, что не отключен системный звук

### Отключение звуков

Просто не указывайте поле `sound_out` в действии:

```json
"open": {
  "type": "exeStart",
  "path": ""
  // sound_out отсутствует — звука не будет
}
```

## 🎤 Настройка микрофона

По умолчанию помощник автоматически выбирает микрофон. Если у вас несколько устройств ввода, вы можете указать конкретный микрофон по его ID.

### Шаг 1: Узнайте ID вашего микрофона

Запустите утилиту для просмотра списка доступных микрофонов:

```bash
# Через виртуальное окружение
.venv\Scripts\python.exe utils\list_mics.py

# Или напрямую через Python
python utils\list_mics.py
```

**Пример вывода:**
```
============================================================
📋 Доступные аудиоустройства
============================================================

🎤 Входные устройства (микрофоны):
------------------------------------------------------------
   ID: 1
   Название: Микрофон (Realtek High Definition Audio)
   Каналы: 2
   Частота: 48000 Гц
------------------------------------------------------------
   ID: 5
   Название: Микрофон (USB Веб-камера)
   Каналы: 1
   Частота: 16000 Гц
------------------------------------------------------------

💡 Чтобы использовать конкретный микрофон:
   1. Скопируйте ID нужного устройства
   2. Откройте config/settings.json
   3. Установите "microphone_id": <ваш_ID>
   4. Сохраните файл и перезапустите помощника
============================================================
```

### Шаг 2: Настройте `config/settings.json`

Откройте файл `config/settings.json` и укажите ID нужного микрофона:

```json
{
  "microphone_id": 5,
  "vad_threshold": 0.05,
  "activation_timeout": 300,
  "active_timeout": 60
}
```

| Параметр | Тип | Описание |
|----------|-----|----------|
| `microphone_id` | integer/null | ID микрофона (из шага 1) или `null` для автовыбора |
| `vad_threshold` | float | Порог чувствительности VAD (0.01–0.5) |
| `activation_timeout` | integer | Таймаут ожидания активационной фразы (секунды) |
| `active_timeout` | integer | Таймаут активного режима (секунды) |

### Шаг 3: Перезапустите помощника

При следующем запуске помощник будет использовать указанный микрофон. В логе вы увидите:

```
🎯 Используем микрофон ID: 5
🎤 Используется устройство ID 5: Микрофон (USB Веб-камера)
```

### Возврат к автовыбору

Чтобы вернуть автоматический выбор микрофона, установите:

```json
{
  "microphone_id": null,
  ...
}
```

Или просто удалите файл `config/settings.json` — будут использованы значения по умолчанию.

## 🔐 Безопасность

### ⚠️ Предупреждения

| Риск | Описание | Митигация |
|------|----------|-----------|
| **Системные команды** | `shutdown`, `logout` могут быть вызваны голосом | Отключите в конфиге или добавьте подтверждение |
| **Запуск приложений** | `exeStart` может запустить любой файл | Валидация путей в `executor.py` |
| **Завершение процессов** | `killProcess` может остановить важные службы | Ограничьте список процессов в конфиге |
| **Нет аутентификации** | Любой голос может отдавать команды | Режим «только доверенные команды» |

### 🔒 Рекомендации

1. **Не запускайте от администратора** — базовые функции работают без прав
2. **Проверьте пути в `commands.json`** — только доверенные приложения
3. **Отключите опасные команды** для продакшена:
   ```json
   "system": {
     "deactivate": { "type": "assistantDeactivate" },
     "stop_assistant": { "type": "assistantStop" }
     // "shutdown", "sleep", "logout" — удалите или закомментируйте
   }
   ```
4. **Добавьте подтверждение** для критичных действий (требует модификации `actions/system.py`)
5. **Регулярно проверяйте логи** на подозрительную активность

## 🔧 Troubleshooting

### Проблема: «firstStart.bat не запускается»

**Решение:**
1. Проверьте версию Python: `python --version`
2. Убедитесь что Python добавлен в PATH (галочка при установке)
3. Запустите от имени пользователя (не администратора)

### Проблема: «Модель не найдена»

**Решение:**
1. Скачайте модель с [vosk-model-small-ru-0.22.zip](https://alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip)
2. Распакуйте в `models/` (должна быть папка `vosk-model-small-ru-0.22/`)
3. Проверьте путь в `audio/recognizer.py`: `DEFAULT_MODEL_PATH`

### Проблема: «start.bat не находит .venv»

**Решение:**
1. Запустите сначала `firstStart.bat` для создания окружения
2. Не удаляйте папку `.venv/` после установки
3. Проверьте что `.venv\Scripts\python.exe` существует

### Проблема: «Микрофон не работает»

**Решение:**
1. Проверьте доступ к микрофону в настройках Windows
2. Убедитесь, что микрофон выбран устройством записи по умолчанию
3. Перезапустите приложение после подключения микрофона
4. **Используйте утилиту выбора микрофона:**
   ```bash
   python utils\list_mics.py
   ```
   Узнайте ID нужного микрофона и укажите его в `config/settings.json`

### Проблема: «Помощник использует не тот микрофон»

**Решение:**
1. Запустите утилиту для просмотра микрофонов:
   ```bash
   python utils\list_mics.py
   ```
2. Найдите в списке нужный микрофон и скопируйте его ID
3. Откройте `config/settings.json`
4. Установите `"microphone_id": <ID_микрофона>`
5. Перезапустите помощника

### Проблема: «Команды не распознаются»

**Решение:**
1. Говорите чётче, делайте паузы между словами
2. Проверьте `config/commands.json` — синонимы должны совпадать
3. Посмотрите логи — текст нормализуется, проверьте `logs/assistant.log`
4. Увеличьте `max_distance` в `correct_text_tokens()` для лучшего исправления опечаток

### Проблема: «Звук не воспроизводится»

**Решение:**
1. Проверьте наличие файла в `sounds/`
2. Убедитесь, что имя в `sound_out` совпадает (регистр важен!)
3. Проверьте системную громкость
4. На Windows: `winsound` может требовать права на воспроизведение

### Проблема: «Окна не управляются»

**Решение:**
1. Установите зависимости: `pip install keyboard pygetwindow`
2. Запустите от имени пользователя (не администратора)
3. Проверьте, что окно активно при команде

## ✨ Возможности

| Категория | Команды | Примеры |
|-----------|---------|---------|
| **Система** | Выключение, сон, выход | «Джарвис, выключи ПК», «переведи в сон» |
| **Окна** | Развёртывание, переключение | «разверни окно», «альт таб» |

### 🔹 Ключевые преимущества

| Особенность | Описание |
|-------------|----------|
| 📶 **Офлайн-работа** | Распознавание через Vosk локально, без отправки данных в облако |
| 🔤 **Исправление опечаток** | RapidFuzz корректирует ошибки распознавания |
| 🎯 **Умный парсинг** | Двухэтапный поиск: сначала многословные фразы, потом токены |
| 🔊 **Звуковая обратная связь** | WAV-файлы для подтверждения каждой команды |
| ⚡ **Два режима работы** | Ожидание (активация по фразе) → Активный (приём команд) |
| 🧩 **Модульная архитектура** | Каждый компонент независим, легко расширять |
| 🎤 **VAD (Voice Activity Detection)** | Энергетический детектор речи отфильтровывает тишину |

### 1. Установка

Запустите **`firstStart.bat`** для первичной настройки:

```batch
firstStart.bat
```

Скрипт автоматически:
- ✅ Проверит версию Python (требуется 3.8+)
- ✅ Создаст виртуальное окружение в `.venv/`
- ✅ Установит все зависимости
- ✅ Создаст папку `models/` для модели распознавания
- ✅ Создаст `config/commands.json` из шаблона

---

### 2. Скачайте модель распознавания

1. Скачайте: [vosk-model-small-ru-0.22.zip](https://alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip)
2. Распакуйте **содержимое архива** в `models/`:

```
models/
└── vosk-model-small-ru-0.22/
    ├── am/
    ├── conf/
    ├── graph/
    └── ...
```

**Важно:** Распаковывайте именно содержимое архива, а не сам архив в папку.

---

### 3. Настройка микрофона

**Шаг 3.1: Узнайте ID микрофона**

Запустите утилиту для просмотра доступных устройств:

```batch
.venv\Scripts\python.exe -m utils.list_mics
```

Вы увидите список всех микрофонов:

```
============================================================
📋 Доступные аудиоустройства
============================================================

🎤 Входные устройства (микрофоны):
------------------------------------------------------------
   ID: 0
   Название: Микрофон (Realtek Audio)
   Каналы: 2
   Частота: 48000 Гц
------------------------------------------------------------
   ID: 1
   Название: Микрофон (USB Headset)
   Каналы: 1
   Частота: 44100 Гц
------------------------------------------------------------
```

**Шаг 3.2: Запишите ID нужного микрофона**

Скопируйте ID устройства (например, `1`).

**Шаг 3.3: Отредактируйте `config/settings.json`**

Откройте файл и укажите ID микрофона:

```json
{
  "microphone_id": 1,
  "vad_threshold": 0.05,
  "activation_timeout": 300,
  "active_timeout": 60
}
```

| Параметр | Значение | Рекомендация |
|----------|----------|--------------|
| `microphone_id` | ID устройства | `0` = автовыбор, или конкретный ID |
| `vad_threshold` | Порог VAD | `0.03`–`0.1` для тихих помещений, `0.1`–`0.2` для шумных |
| `activation_timeout` | Таймаут активации | `300` сек (5 мин) |
| `active_timeout` | Таймаут активного режима | `60` сек |

### ➕ Как добавить команду

**5 шагов:**

1. **Добавьте идентификатор:**
   ```json
   "identifiers": {
     "discord": ["дискорд", "дс"]
   }
   ```

2. **Добавьте глагол (если нужен новый):**
   ```json
   "verbs": {
     "shuffle": ["перемешай", "случайный порядок"]
   }
   ```

3. **Опишите действие:**
   ```json
   "actions": {
     "discord": {
       "open": {
         "type": "exeStart",
         "path": "C:\\Users\\User\\AppData\\Local\\Discord\\Update.exe",
         "sound_out": "ok1.wav"
       }
     }
   }
   ```

4. **Добавьте звук** в `sounds/` (опционально)

5. **Перезапустите** помощника

## 🔊 Звуковая обратная связь

Каждое действие может иметь звуковой отклик через поле `sound_out`:

```json
"browser": {
  "open": {
    "type": "exeStart",
    "path": "...",
    "sound_out": "ok1.wav"
  }
}
```

### Доступные звуки

| Файл | Назначение |
|------|------------|
| `ok1.wav`, `ok2.wav`, `ok3.wav`, `ok4.wav` | Подтверждение выполнения |
| `reply1.wav` – `reply6.wav` | Ответы ассистента |
| `greet1.wav` | Приветствие |
| `thanks.wav` | Благодарность |
| `chord.wav`, `SWO.wav` | Эффекты |

**Требования к файлам:**
- Формат: WAV (PCM, 16-bit, 44.1kHz)
- Расположение: папка `sounds/`
- Размер: до 500 КБ

**Отключение звука:** не указывайте поле `sound_out`.

### Расширение: новый тип действия

**1. Создайте исполнитель в `actions/my_action.py`:**

```python
from actions.base import ActionExecutor
from typing import Dict, Any

class MyActionExecutor(ActionExecutor):
    def execute(self, action_def: Dict[str, Any]) -> bool:
        # Ваша логика
        success = True
        sound_out = action_def.get("sound_out")
        if sound_out:
            self.play_sound(sound_out)
        return success

    def get_action_type(self) -> str:
        return "myNewAction"
```

**2. Зарегистрируйте в `actions/__init__.py`:**

```python
from actions.my_action import MyActionExecutor

class ActionDispatcher:
    def __init__(self, sound_player=None):
        self._executors = {
            "my_action": MyActionExecutor(sound_player),
        }
        self._action_type_map = {
            "myNewAction": "my_action",
        }
```

**3. Используйте в `commands.json`:**

```json
"myFeature": {
  "activate": {
    "type": "myNewAction",
    "sound_out": "feature.wav"
  }
}
```

## 🔧 Troubleshooting

### 🔴 Микрофон не найден

**Ошибка:**
```
[ERROR] Не удалось получить доступ к микрофону
💡 Попробуйте перезапустить программу через 30 секунд
```

**Причины и решения:**

| Причина | Решение |
|---------|---------|
| Микрофон не подключён | Проверьте подключение устройства |
| Неверный ID в `settings.json` | Запустите `.venv\Scripts\python.exe -m utils.list_mics` и укажите правильный ID |
| Доступ запрещён | Разрешите доступ к микрофону в настройках Windows |
| Устройство занято | Закройте другие программы (Discord, Zoom, браузер) |
| Драйверы устарели | Обновите аудиодрайверы с сайта производителя |

**Пошаговое решение:**

1. **Проверьте доступность микрофона в Windows:**
   - `Параметры` → `Конфиденциальность` → `Микрофон`
   - Убедитесь, что «Доступ к микрофону» включён
   - Проверьте, что приложениям разрешён доступ

2. **Узнайте ID микрофона:**
   ```batch
   .venv\Scripts\python.exe -m utils.list_mics
   ```

3. **Отредактируйте `config/settings.json`:**
   ```json
   {
     "microphone_id": 1
   }
   ```

4. **Перезапустите помощника через 30 секунд**

### 🔴 Не тот микрофон (шум, тихий звук)

**Симптомы:**
- Помощник не реагирует на речь
- Срабатывает на фоновый шум
- Распознавание некорректное

**Решение:**

**Шаг 1: Определите правильный микрофон**

```batch
.venv\Scripts\python.exe -m utils.list_mics
```

Выберите устройство с нужным названием (например, `USB Headset` вместо `Realtek Audio`).

**Шаг 2: Настройте VAD (Voice Activity Detection)**

Откройте `config/settings.json`:

```json
{
  "microphone_id": 1,
  "vad_threshold": 0.05
}
```

| Значение `vad_threshold` | Когда использовать |
|--------------------------|-------------------|
| `0.01` – `0.03` | Очень тихая комната, чувствительный микрофон |
| `0.03` – `0.07` | Обычные условия (рекомендуется) |
| `0.07` – `0.15` | Шумное окружение, вентилятор, клавиатура |
| `0.15` – `0.3` | Очень шумно, несколько источников звука |

**Шаг 3: Откалибруйте VAD во время работы**

Запустите помощника, скажите активационную фразу, затем:

```
«калибруй шум»
```

Помощник запишет 3 секунды фонового шума и автоматически настроит порог.

**Шаг 4: Проверьте расстояние до микрофона**

- Оптимальное расстояние: 10–30 см от рта
- Избегайте направления на источники шума (вентилятор, клавиатура)

**Ошибка:**


**Решение:**

1. Скачайте модель: [vosk-model-small-ru-0.22.zip](https://alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip)

2. Распакуйте **содержимое архива** в `models/`:
   ```
   models/
   └── vosk-model-small-ru-0.22/
       ├── am/
       ├── conf/
       ├── graph/
       └── ...
   ```

3. Проверьте структуру:
   ```
   models/vosk-model-small-ru-0.22/conf/model.conf  ← должен существовать
   ```

4. Перезапустите помощника

### 🔴 Команда не распознаётся

**Симптомы:**
- Помощник активируется, но не выполняет команду
- В логе: `Команда не распознана`

**Причины и решения:**

| Причина | Проверка | Решение |
|---------|----------|---------|
| Нет в `commands.json` | Проверьте `identifiers` и `verbs` | Добавьте команду |
| Опечатка в распознавании | Смотрите лог `logs/assistant.log` | RapidFuzz исправит при `max_distance=2` |
| Неправильный порядок слов | Попробуйте разные формулировки | Используйте многословные фразы |
| Конфликт команд | Проверьте логи | Уточните формулировку |

**Диагностика:**

1. Откройте `logs/assistant.log`
2. Найдите строки с `🎤 Распознано:`
3. Сравните распознанный текст с командами в `commands.json`

**Пример:**
```
🎤 Распознано: 'аткрой браyзер' → 'аткрой браyзер'
🔧 Исправлено: 'аткрой браyзер' → 'открой браузер'
🎯 Команда распознана: browser/open
```

### 🔴 Звук не воспроизводится

**Симптомы:**
- Команда выполняется, но звука нет

**Причины и решения:**

| Причина | Решение |
|---------|---------|
| Файл не найден | Проверьте наличие в `sounds/` |
| Неверное имя в `sound_out` | Имя должно точно совпадать (регистр!) |
| Системная громкость | Проверьте микшер громкости Windows |
| Файл повреждён | Откройте в медиаплеере для проверки |

**Проверка:**

1. Убедитесь, что файл существует:
   ```
   sounds/ok1.wav  ← должен быть
   ```

2. Проверьте имя в `commands.json`:
   ```json
   {
     "sound_out": "ok1.wav"
   }
   ```

3. Откройте файл в любом плеере (VLC, Windows Media Player)

### 🔴 Помощник не активируется

**Симптомы:**
- Говорите активационную фразу, но реакции нет

**Причины:**

| Причина | Решение |
|---------|---------|
| VAD слишком высокий | Уменьшите `vad_threshold` в `settings.json` |
| Тихая речь | Говорите громче, ближе к микрофону |
| Фоновый шум | Уменьшите шум, настройте VAD |
| Неверная активационная фраза | Проверьте `activation_phrases` в `commands.json` |

**Настройка VAD:**

```json
{
  "vad_threshold": 0.03
}
```

**Проверка активационных фраз:**

Откройте `config/commands.json`:

```json
{
  "activation_phrases": ["джарвис", "эй помощник", "слушай"]
}
```

Добавьте свои варианты при необходимости.

### 🔴 Частые вылеты / ошибки Python

**Ошибка импорта:**
```
ModuleNotFoundError: No module named 'vosk'
```

**Решение:**
```batch
.venv\Scripts\activate
pip install -r requirements.txt
```

**Ошибка PyAudio:**
```
ModuleNotFoundError: No module named 'pyaudio'
```

**Решение:**
```batch
pip install pipwin
pipwin install pyaudio
```

**Общая ошибка:**
```
ImportError: DLL load failed
```

**Решение:**
1. Переустановите виртуальное окружение:
   ```batch
   rmdir /s /q .venv
   python -m venv .venv
   .venv\Scripts\activate
   pip install --upgrade pip
   pip install -r requirements.txt
   ```
